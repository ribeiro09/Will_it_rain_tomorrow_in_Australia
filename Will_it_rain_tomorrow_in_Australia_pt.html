Rain in Australia¶

Esse projeto para fazer um modelo de predio para previso do tempo para o proxmo dia na Australia, se vai chover ou no, atravs de um resultado binrio em um sistema treinado.

Conteúdo¶

Esse dataset contem dados de dias obeservados e registrados por diversas estaes meteorolgicas da Australia.

O objetivo a varivel RainTomorrow, que reponde a pergunta: "Vai chover amanh? Sim ou No".

Fontes¶

Essa fonte de dados se encontra na plataforma Kaggle em: https://www.kaggle.com/jsphyg/weather-dataset-rattle-package/home. Para mais detalhes para a fonte de dados acesse o link.

Vamos remover os outliers usando a tecnica Z-score. Essa tcnica faz o escalonamento de todos os valores do dataframe e os relaciona com a mdia e o desfio padro, gerando um score para cada valor. Assim os valores que tiverem scores prximos de 3 e -3 so considerados outliers, podendo ser removidos do dataframe.

A primeira estapa do processo vai ser determinar quais so as colunas (vou vomear a chamar de features daqui para frente, termo usado em ingls) tem maior relevncia para o medelo preditivo. Para isso vou aplicar a ferramenta SelectKBest para gerar uma pontuao (score) das features para ver quais so mais relevantes.

Outra mudana vai ser chamar os resultados da predio de labels, que o objetivos dos modelos, determinar se chove ou no amanh.

# Criando listas para armazernar os resultados de cada modelo
# Para Logistic Regression
accuracy_LR=[]
# Para Decision Tree
accuracy_dt=[]
# Para Kmeans
accuracy_Kmeans=[]

# Criando o loop on n  o nmero de features na lista contadore de features
for n in n_features_list:  
    
    # Separando os valores para os conjuntos de treinamento e teste com "train_test_split", vamos deixar 20% dos dados
    # para teste e o resto para treinamento.
    features_train, features_test, labels_train, labels_test = train_test_split(df[K_values[:n]], y, test_size=0.2, random_state=42)

    # Chamando a funo do mtodo de Logistic Regression
    l_clf = LogisticRegression()
    # Treinando
    l_clf.fit(features_train, labels_train)
    # Fazendo a predio com o grupo de valores de teste
    prediction_lr = l_clf.predict(features_test)
    # Armazenando o valor da accuracy na lista
    accuracy_LR.append(accuracy_score(labels_test, prediction_lr))
    
    # Os mesmos passos so apicados para os outros dois mtodos, na mesma ordem
    
    # Para Decision Tree
    dt_clf = DecisionTreeClassifier(random_state=0)
    dt_clf.fit(features_train, labels_train)
    prediction_dt = dt_clf.predict(features_test)
    accuracy_dt.append(accuracy_score(labels_test, prediction_dt))
    
    # Para Kmeans
    k_clf = KMeans(n_clusters=2)
    k_clf.fit(features_train, labels_train)
    prediction_k = k_clf.predict(features_test)
    accuracy_Kmeans.append(accuracy_score(labels_test, prediction_k))

O modelo de validao consiste em fazer mil interaes dos modelos preditivos e retornar a accuracy, precision e recal para uma lista. Apartir dos resultados fazer uma mdia dos valores para possuir um valor mais real dos parmetos de avaliao para determinar o melhor modelo. O que permite fazer essas mil interaes para validar os modelos que a funo train_test_split vai selecionar em cada interao um conjuto de dados diferente para teste e treinamento, assim podendo cobrir um quantidade maior de formaes para que o modelo treine e gere resultados mais confiveis, uma vez cobrindo combinaes unicas.

Os parmetro precision retorna a proporo de verdadeiros positivos nos registros de predio de chuva e descrevendo a frenquncia de falsos negativos. J recall demostra a proporo de verdadeiros positivos nos resultados de predio reais de chuva e descrevendo a sensibilidade dos resultados. Esse dois mtodos podem ajudar melhor em mdotos de predio binrios que somente accuracy evitando overfitings.
